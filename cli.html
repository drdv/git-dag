Command-line interface¶

Visualize the git DAG.

usage: git-dag [-h] [-p PATH] [-f FILE] [-b {graphviz}] [--format FORMAT]
               [--dpi DPI] [-i INIT_REFS [INIT_REFS ...]]
               [-R RANGE [RANGE ...]] [-n MAX_NUMB_COMMITS]
               [--rankdir {LR,RL,TB,BT}] [--bgcolor BGCOLOR] [-u] [-t] [-D]
               [-l] [-r] [-s] [-H] [-T] [-B] [-m COMMIT_MESSAGE_AS_LABEL] [-o]
               [--log-level {NOTSET,INFO,WARNING,ERROR,CRITICAL}]

Named Arguments¶

-p, --path

Path to a git repository.

Default: '.'

-f, --file

Output graphviz file (e.g., /path/to/file).

Default: 'git-dag.gv'

-b, --dag-backend

Possible choices: graphviz

Backend DAG library.

Default: 'graphviz'

--format

Graphviz output format (tooltips are available only with svg). If the format is set to ‘gv’, only the graphviz source file is generated.

Default: 'svg'

--dpi

DPI of output figure (used with --format png).

-i, --init-refs

A list of branches, tags, git objects (commits, trees, blobs) that represents a limitation from where to display the DAG.

-R, --range

A list to commits in a range to display.

-n, --max-numb-commits

Max number of commits (set to 0 to remove limitation).

Default: 1000

--rankdir

Possible choices: LR, RL, TB, BT

rankdir argument of graphviz.

Default: 'TB'

--bgcolor

bgcolor argument of graphviz (e.g., transparent).

Default: 'gray42'

-u

Show unreachable commits.

Default: False

-t

Show tags.

Default: False

-D

Show deleted annotated tags.

Default: False

-l

Show local branches.

Default: False

-r

Show remote branches.

Default: False

-s

Show stash.

Default: False

-H

Show head.

Default: False

-T

Show trees (WARNING: should be used only with small repositories).

Default: False

-B

Show blobs (discarded if -T is not set).

Default: False

-m, --message

When greater than 0, this is the number of characters from the commit message to use as a commit label. The commit SHA is used otherwise.

Default: 0

-o, --xdg-open

Open output file with xdg-open.

Default: False

--log-level

Possible choices: NOTSET, INFO, WARNING, ERROR, CRITICAL

Log level.

Default: 'WARNING'
